## Ways of working
- **Version control:** Git + GitHub; feature branches → pull request → review → merge.
- **Tracking:** Trello board (see `Trello_Board.md`), short daily check-ins.
- **Definition of Done:** code documented, tested where relevant, reviewed by another member, merged.
- **Cost control:** billable API calls cached (SQLite TTL); tests run on fallbacks.
